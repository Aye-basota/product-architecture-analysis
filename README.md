# Product Architecture & Tech Roles Analysis

A personal write-up exploring how real-world software products are structured, what engineering roles are involved, and how to map personal skills to market demands.

## Topics covered

- Software product architecture: components, data flow, deployment
- Tech roles: backend, frontend, DevOps, data, QA, product
- Git and GitHub workflow for collaboration
- Personal skill gap analysis and growth planning

- `diagrams/` — system diagrams
- `images/` — visuals and screenshots

Written as a structured exercise to build fluency in describing software systems and reflecting on career trajectory in tech.
